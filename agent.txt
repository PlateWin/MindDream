这是一个 MADAS (Multi-Agent Dream Analysis System) v2.0 "Full Spectrum" 的终极设计方案。

既然不再受数量限制，我们将采用 「流水线 + 专家委员会 + 陪审团」 的全链路架构。这套系统共包含 5 个阶段，由 28 个专用 Agent 组成。

本设计文档严格遵循软件工程规范，定义了每个 Agent 的输入/输出接口 (I/O Interface) 和核心职责。

🏗️ 系统拓扑图 (System Topology)
Phase 1: 预处理与净化 (The Airlock) - 2 Agents

Phase 2: 结构化取证 (Forensic Analysis) - 5 Agents

Phase 3: 诠释委员会 (The Council of Interpretation) - 12 Agents (分为4个学派小组)

Phase 4: 认知对抗与审计 (Adversarial Audit) - 5 Agents

Phase 5: 综合与交付 (Synthesis & Delivery) - 4 Agents

🟢 Phase 1: 预处理与净化 (The Airlock)
目标：将非结构化的用户胡言乱语转化为安全的、机器可读的标准格式。

01. Privacy_Masker (隐私面具师)
作用: 彻底的 PII (Personally Identifiable Information) 清洗。

输入: 用户原始文本 (Raw Text)。

输出: 脱敏文本 (Sanitized Text) + 替换映射表 (Mapping Dict)。

逻辑: 使用 NER (命名实体识别)。"我梦见和李明在天安门吵架" -> "我梦见和[Person_A]在[Landmark_B]吵架"。

02. Narrative_Linearizer (叙事线性化器)
作用: 修复逻辑断裂，将倒叙、插叙统一为线性时间轴。

输入: 脱敏文本。

输出:

JSON
{ "timeline": [
    {"seq": 1, "action": "walking", "context": "forest"}, 
    {"seq": 2, "action": "falling", "context": "cliff"}
]}
逻辑: 补全省略的主语，明确动作的发起者和承受者。

🔵 Phase 2: 结构化取证 (Forensic Analysis)
目标：不进行解释，只提取“梦境的物理参数”。

03. Entity_Extractor (实体提取器)
作用: 提取梦中出现的所有物体、人物、地点。

输入: 线性时间轴。

输出: Entity_List (e.g., ["Snake", "Teacher", "HighSchool"])。

04. Sensory_Modality_Scorer (感官模态评分员)
作用: 分析五感权重。痛觉和强烈的冷热感通常指向生理刺激而非心理隐喻。

输入: 脱敏文本。

输出: Sensory_Vector (e.g., {Visual: 0.9, Auditory: 0.2, Pain: 0.0, Tactile: 0.5})。

05. Sentiment_Seismograph (情绪地震仪)
作用: 绘制情绪起伏曲线。

输入: 线性时间轴。

输出: Emotion_Curve (e.g., Start: Anxiety(0.4) -> Middle: Terror(0.9) -> End: Relief(0.2))。

06. Bizarreness_Detector (怪诞探测器)
作用: 标记物理法则失效的时刻（如飞行、变形）。这是区分 REM 梦境与 NREM 梦境的关键。

输入: 线性时间轴。

输出: Reality_Breaches List (e.g., ["Gravity failure at seq 3", "Object transformation at seq 5"])。

07. Day_Residue_Scanner (日间残留扫描仪)
作用: 区分“记忆回放”与“梦境创造”。需要联网搜索或访问用户近期日志。

输入: Entity_List + 用户近期上下文 (User Context)。

输出: Residue_Flagged_Entities (e.g., {"Snake": "Saw in movie yesterday - LOW SYMBOLISM"})。

🟣 Phase 3: 诠释委员会 (The Council)
目标：从不同理论视角生成互斥的假设。

[Group A: 心理动力学派]
08. Freudian_Decoder (弗洛伊德代理)
作用: 寻找愿望满足与性/攻击本能的伪装。

输入: Entity_List (除去日间残留)。

输出: Latent_Content_Hypothesis (e.g., "Sword represents phallic symbol/power struggle").

09. Jungian_Archetypist (荣格代理)
作用: 寻找集体潜意识原型 (Shadow, Anima/Animus, Self)。

输入: Entity_List + Narrative。

输出: Archetype_Mapping (e.g., "Old man represents The Wise Old Man archetype").

10. Gestalt_Roleplayer (完形流派代理)
作用: 假设梦中每个物体都是做梦者人格的一部分。

输入: Entity_List。

输出: Part_Self_Dialogues (e.g., "If you are the snake, what do you want to say to the dreamer?").

[Group B: 认知与进化派]
11. Threat_Simulator (威胁仿真代理)
作用: 基于进化心理学 (Revonsuo)，寻找生存技能训练的迹象。

输入: Timeline + Emotion_Curve (Focus on Negative emotions).

输出: Survival_Training_Score (e.g., "Chasing sequence identified as predator avoidance training").

12. Memory_Consolidator (记忆整合代理)
作用: 分析梦境如何连接新旧记忆。

输入: Day_Residue + Timeline.

输出: Memory_Linkage_Map。

13. Metaphor_Linguist (隐喻语言学家)
作用: 基于 Lakoff 的具身认知理论，将动作转化为语言隐喻。

输入: Action_List。

输出: Linguistic_Metaphors (e.g., "Falling -> Falling out of favor / Loss of control").

[Group C: 生理与虚无主义派]
14. Stochastic_Noise_Generator (随机噪声代理)
作用: 基于 Hobson 的激活-综合假说，主张梦无意义。

输入: Reality_Breaches.

输出: Randomness_Assertations (e.g., "Visual chaos is likely PGO waves from brainstem, no meaning").

15. Somatic_Diagnostician (躯体诊断代理)
作用: 检查梦境是否反映身体状态（如疾病前兆、睡眠姿势）。

输入: Sensory_Vector.

输出: Somatic_Alerts (e.g., "Teeth falling out relates to teeth grinding (Bruxism) probability 60%").

[Group D: 文化与灵性派]
16. Cultural_Symbolist (文化符号学家)
作用: 基于用户文化背景（东亚/西方/其他）查询符号库。

输入: Entity_List + User_Culture_Tag.

输出: Cultural_Interpretation.

🔴 Phase 4: 认知对抗与审计 (Adversarial Audit)
目标：清洗、驳斥、去伪存真。

17. Debate_Moderator (辩论主持人)
作用: 发现各学派之间的最大矛盾点。

输入: Phase 3 所有 Agent 的输出。

输出: Conflict_Pairs (e.g., "Freud says X, but Neuroscience says Y").

18. Barnum_Filter (巴纳姆过滤器)
作用: 识别并删除模棱两可的废话（“你外表坚强内心柔软”）。

输入: 所有解释文本。

输出: Redacted_Text (删除了废话的版本).

19. Overfitting_Watchdog (过拟合看门狗)
作用: 防止对微小细节的过度解读。

输入: 解释文本长度 vs 原始文本长度。

输出: Truncation_Orders (截断指令).

20. Safety_Compliance_Officer (合规官)
作用: 拦截诱导自杀、犯罪、医疗诊断的内容。

输入: 净化后的解释。

输出: Safety_Flag (Pass/Block).

21. Confidence_Scorer (置信度打分员)
作用: 为每个保留下来的观点打分。

输入: 解释文本 + 证据链。

输出: Weighted_Interpretations (e.g., {"Metaphor": 0.85, "Freud": 0.3}).

🟡 Phase 5: 综合与交付 (Synthesis & Delivery)
目标：生成最终的用户界面数据。

22. Insight_Synthesizer (洞察合成器)
作用: 将加权后的解释融合成连贯的段落，使用“多重视角”句式。

输入: Weighted_Interpretations.

输出: 核心分析报告 (Core Analysis Report).

23. Question_Generator (探询生成器)
作用: 生成 3-5 个引导用户反思的问题，而不是给答案。

输入: 核心冲突点。

输出: Reflection_Questions.

24. Visual_Data_Mapper (可视化数据映射)
作用: 生成前端图表所需的 JSON。

输入: Emotion_Curve, Sensory_Vector, Confidence_Scores.

输出: Dashboard_JSON.

25. Actionable_Advice_Provider (行动建议者)
作用: 仅基于 CBT (认知行为疗法) 和睡眠卫生学提供建议。

输入: 情绪基调。

输出: Action_Items (e.g., "Write a dream journal", "Progressive Muscle Relaxation").

🧩 完整的 JSON 数据流示例
为了让这个多智能体系统真正跑起来，你需要定义一个全局的 State Object 在这些 Agent 之间流转：

JSON
{
  "request_id": "dream_12345",
  "raw_input": "我梦见牙齿全碎了...",
  "meta_data": {
    "user_id": "u_001",
    "culture": "East_Asian"
  },
  "pipeline_stages": {
    "sanitized": {
      "text": "...",
      "entities": ["Teeth", "Mirror"]
    },
    "forensics": {
      "bizarreness_score": 0.2,
      "dominant_emotion": "Fear",
      "is_day_residue": false
    },
    "interpretations": {
      "psychoanalysis": "Loss of power...",
      "neuroscience": "Somatic signal...",
      "cultural": "Bad omen regarding parents (Filtered by Safety Agent)"
    },
    "audit": {
      "removed_statements": ["You will lose money soon"],
      "confidence_map": {"somatic": 0.8, "symbolic": 0.4}
    }
  },
  "final_output": {
    "report_markdown": "...",
    "charts": "..."
  }
}
这套 25+ Agent 的架构不再是简单的聊天机器人，它是一个仿真的人类专家会诊小组。每个 Agent 只需要一个很小的、很专注的 System Prompt 即可高质量完成任务。
