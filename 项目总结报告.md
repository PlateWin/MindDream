## 一、项目概述

### 1.1 项目背景

**MindDream** 是一个专业的心理学梦境数据分析平台，致力于通过 AI 技术帮助用户探索梦境背后的潜意识状态。平台集成了梦境数据管理、多维度可视化统计、智能 AI 分析和对话功能，旨在为每一个灵魂架起通往潜意识世界的桥梁。

### 1.2 核心目标

| 目标维度 | 具体指标 | 完成状态 |
|----------|----------|----------|
| 🧠 情绪分类 | 对梦境情绪做 6 分类，准确率目标 ≥ 70% | ✅ v1.0 基础模型就绪 |
| 🔒 隐私保护 | 确保用户数据 100% 合规存储 | ✅ MIT 开源，数据脱敏 |
| 🚀 开源社区 | 项目发布至 GitHub，目标 ≥ 50 Stars | ✅ 仓库已建立 |
| 📊 可视化洞察 | 生成梦境中的潜意识关键词云、潜在示例 | ✅ ECharts 多维图表 |

### 1.3 项目里程碑

```
2025.12  Phase 1 · 需求分析与技术选型
         PRD 文档 / 高保真原型 / 系统架构图 / Dify 初始化
```

## 五、上线与部署

### 5.1 生产环境部署方案

#### 部署架构

```
互联网 → Nginx (:80/:443) → 前端静态文件 (dist/)
                           → /api/* 代理 → Spring Boot (:8080) → MySQL (:3306)
```

#### 部署步骤

1. **数据库部署**：MySQL 8.0 安装 → 建库 `minddream` → 执行 `schema.sql` → 导入 JSONL 数据
2. **后端打包部署**：`mvn clean package -DskipTests` → `java -jar minddream-backend-1.0.0.jar`（支持 systemd 服务化）
3. **前端打包部署**：`npm run build` → `dist/` 部署至 Nginx
4. **Nginx 配置**：SPA 路由 fallback + `/api/` 反向代理 + 静态资源缓存
5. **安全加固**：HTTPS（Let's Encrypt）、数据库权限最小化、防火墙配置

## 九、经验总结与展望

### 9.1 项目亮点

1. **完整的技术栈覆盖**：Spring Boot + Vue 3 前后端分离架构，从数据层到展示层全链路打通。
2. **Apple Liquid Glass UI**：独特的液态玻璃设计风格，Morphing Blob 背景 + 3D 磁力倾斜 + 多主题系统，用户体验出色。
3. **文档体系完善**：12 份文档覆盖入门、开发、部署、运维全流程，新人可快速上手。
4. **开源友好**：MIT 许可证、清晰的项目结构、中英文文档，降低社区参与门槛。
5. **AI 集成**：梦境综合解读、情感分析、符号解读、个人洞察、AI 对话五位一体的智能分析能力。

### 9.2 待改进项

| 问题 | 影响 | v2.0 计划 |
|------|------|-----------|
| 用户系统缺失 | 当前无登录/注册，所有功能公开访问 | Phase 2 补全 |
| NLP 模型准确率 | 当前基础模型准确率约 60%，距目标 70% 有差距 | BERT/RoBERTa 微调 |
| 安全加固不足 | SQL 注入/XSS 基础防护已完成，生产级加固待加强 | Phase 5 |
| CI/CD 未建立 | 当前手动部署，缺少自动化流水线 | Phase 5 |
| 单元测试覆盖率 | 后端缺少系统化单元测试 | Phase 5 QA |
| GitHub Wiki | 已规划但未完成 | 持续建设 |

### 9.3 v2.0 展望

```
技术升级
├── 前端 → React + Tailwind CSS + ECharts/D3.js
├── 算法 → PyTorch + BERT/RoBERTa 微调（6 分类，目标 70%+）
├── AI  → Dify 工作流编排 + 知识库 + AI 对话
└── 可视化 → Stable Diffusion / ControlNet（梦境图片生成，可选）

功能迭代
├── 用户系统（登录/注册/个人中心）
├── 梦境录入与个人梦境历史
├── 关键词提取（TF-IDF / TextRank）
├── 历史记录对比分析
└── 社区功能（梦境分享、匿名讨论）

工程质量
├── CI/CD 流水线（GitHub Actions）
├── 自动化测试（Cypress / Playwright E2E）
├── 安全加固（完整 SQL 注入/XSS/CSRF 防护）
└── 性能优化（缓存层、CDN、数据库调优）
```
